async4j

Callback based asynchronous programming library to improve your application responsiveness and make efficient use of your system resources

Callback but no hell

async4j implement callback for you, just focus on application logic you implement in asynchronous functions that take callback as parameter. So no business logic in callback object! A Fluent APIs that provides asynchrnous equivalent of flow of controls such as if-then-else, while, do-while, foreach or try-catch-finally should be used to compose asynchronous functions.

Support for JVM languages

The core callback API is implemented in Java and can be used as is from any JVM language, whereas Fluent API is more specific to language. A builtin Fulent API is provided for Java language supporting Java 5 and ready for the upcomming Java 8. A Scala dedicated Fluent API is also provided to take advantage of the language expressiveness and features like tuples, for-comprehension, or continuations.

Efficient use of system resources

By asynchronous call you can avoid some blocking operation to improve responsiness of the system. In addition, async4j provides flexibilty to define the efficient execution topology by parallelize non dependent tasks on multi-core architecture or on remote machine easily.
