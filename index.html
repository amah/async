async4j

Callback based asynchronous programming library to improve your application responsiveness and make efficient use of your system resources

Callback but no hell

async4j library make extensive use of callback objects but only for internal implementation of asynchronous flow logics such as if-then-else, loops, try-catch-finally etc... Application logics are to be implemented in composable asynchronous functions which plain functions with one callback object as parameter.

Support for JVM languages

Fluent APIs are provided to adapt callback implementing asynchronous flow logics to the target the language. The built-in Fluent API supports Java 5 or later and use with lambda expressions included in the upcoming Java 8. A separate Fluent API is also provided for Scala to take advantage of the language expressiveness and features like tuples, for-comprehension, or continuations.

Efficient use of system resources

Using asynchronous programming model, you can improve your application responsiveness. In addition, async4j library provides you with facilities to parallelize efficiently non dependent tasks to harness you system resources (CPU, Caches, I/O Bandwidth, memories ... ) especially on multi-core architecture or in distributed environment.
